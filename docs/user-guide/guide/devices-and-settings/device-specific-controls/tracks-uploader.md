# Tracks uploader

## Purpose

The **tracks uploader** lets you import **recorded track / datalogger files** that were exported from a device — useful when a device logged data offline (out of coverage) and you want that history on the platform.

<!-- SCREENSHOT: Tracks upload block — file picker / attach-file control. Annotate: attach-file button, supported formats. -->

## Settings

* **Attach file** — select the recorded track file to upload. Common formats include **CSV, TXT, Excel, and OCT**.

## Appears when

Appears on devices that produce uploadable datalogger/track files.

## Gotchas

* This is a **one-off file upload**, not a stored setting — there is nothing to save on the device.
* The accepted file formats and structure depend on the device's datalogger export.
