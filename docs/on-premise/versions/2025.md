## 2025.11 (17 Nov 2025)

In build 2025.11, we added one more 3rd-party solution for video telematics, along with usual device integrations. We are also intoducing a completely new feature for on-premise: IoT Logic can now be enabled on standalones! Read the [description below](2025.md#iot-logic-support-on-demand) to learn more about this powerful data processing automation tool.

As usual, we continuously work with customer feedback and provide issue fixes and platform improvements.&#x20;

#### **IoT Logic support (on demand)**

No-code/low-code data management tool for processing telematics data from GPS trackers, dash cams, and IoT sensors, previously available only in cloud PaaS and now enabled for standalone deployments. The tool addresses key telematics challenges:

* **Protocol complexity**: [Navixy Generic Protocol](https://app.gitbook.com/s/tx3J5BxnWyPV0nP2xr0z/technologies/navixy-generic-protocol) eliminates custom decoder development and maintenance for different device manufacturers
* **Hardware limitations**: Calculations using [Navixy IoT Logic Expression Language](https://app.gitbook.com/s/tx3J5BxnWyPV0nP2xr0z/technologies/navixy-iot-logic-expression-language) (JEXL-based) derive new metrics from existing sensor data without equipment upgrades
* **Manual data processing**: Visual flow architecture automates repetitive workflows like unit conversions and data validation
* **Device troubleshooting**: [Data Stream Analyzer](https://www.navixy.com/iot-logic/data-stream-analyzer/) enables quick diagnosis of connectivity and transmission issues without vendor support
* **Multi-platform integration**: Configurable output endpoints transmit standardized data to multiple third-party systems simultaneously

To request access, contact your Customer Success Manager or [support@navixy.com](mailto:support@navixy.com).
