# Completed with errors status (Field Service)

## Question

Task is marked as **Completed with errors**, but all subtasks are complete without errors, so we can't find why the main task is marked as with errors. Can you please help us understand it?

## Answer

This status is assigned when an employee completes a route task but visits the checkpoints in the wrong order.

For example, the planned sequence was to visit Point 1, then Point 2, and finally, Point 3, but in reality, the employee visited Point 1 first and then Point 3, only traveling to Point 2 later.

That's why the status is **Completed with errors**. It refers specifically to the order of execution.
