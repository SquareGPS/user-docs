# Incorrect Duration due to invalid date or time format (Tasks import)

### Question

Why does the _duration_ header appear incorrect when I upload the tasks file, requiring me to enter it manually?

### Answer

When the duration appears incorrectly, it suggests that the task start or end date may contain errors that the platform is unable to recognize as valid. As a result, the duration must be entered manually.

In some cases, users include incorrect symbols (for example, 01-02-2026) or time formats such as AM/PM. This causes the platform to interpret the time as invalid, so we recommend carefully reviewing and correcting the date and time fields.
